Intelligence Explainer

Everything you need to know about the 3rdAI Video Analytics Platform — Explained simply.

🎯

What is this Project?

3rdAI is a high-performance AI Traffic Enforcement & Monitoring System. It uses computer vision to automatically detect road violations and identify vehicles from live camera feeds or video files.

⚙️

How does it work?

The system watches a video feed frame-by-frame. When it sees a vehicle, it applies specialized AI models to check for violations like missing helmets or seatbelts. It then captures a photo (crop) and saves the data to the cloud.

🛠️

What is used?

Built with FastAPI (Speed), YOLOv8 (Brain/AI), OpenCV (Vision), PostgreSQL (Memory), and Cloudflare R2 (Evidence Storage). We use WebSockets for real-time live streaming.

🧠

Which model is used?

We use multiple YOLOv8 Precise Models:
• ANPR.pt for Plates
• helmet.pt for Security
• triple.pt for Riders
• seatbelt.pt for Drivers

❓

Why use this?

Standard cameras just record; this camera understands. It removes the need for manual monitoring, detects violations 24/7 with 95%+ accuracy, and provides instant searchable digital records.

📍

Where is it used?

Ideal for Smart Cities, Highway Toll Gates, Private Society Entrances, and Traffic Law Enforcement Agencies who want to automate surveillance and penalties.

Live Detection & Debug Flow

How the system "thinks" and how you can track its actions:

  • 1

    Camera Feed Connection

    The system tries connecting via TCP first (for stability), then UDP (for speed). If it fails, it retries automatically.

    DEBUG: Trying connection via FFmpeg TCP... SUCCESS!
  • 2

    Real-time Tracking

    As frames come in, YOLOv8 assigns a "Track ID" to every vehicle so we don't count the same car twice.

    DEBUG: Loaded model ANPR.pt for Number Plate Detection
  • 3

    Trigger & OCR Detection

    When a plate is found, the system sends a high-res crop to the OCR engine. If the plate matches Indian standards (Regex), it proceeds.

    DEBUG: Valid Plate Result: MH12AB1234
  • 4

    Evidence Persistence

    The image is uploaded to R2 Cloud and the text is saved to our central SQL database for long-term storage.

    DEBUG: SUCCESS! Saved to R2: pub-x.r2.dev/ANPR_123.jpg
  • 5

    Web Feedback (UI)

    The frontend browser receives a WebSocket event and updates the "Activity Log" instantly without refreshing the page.
